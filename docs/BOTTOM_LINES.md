# Bottom Lines

> 本內容僅供研究、教育與娛樂用途，不構成投資建議。

## 最高優先級規則

1. Codex 只能修改 `Minotaur-Grand-Mentor/` 專案資料夾內的檔案。
2. 如果此資料夾位於 Obsidian vault 內，不可以修改任何其他 Obsidian 內容。
3. 不可以刪除或覆蓋既有 `records/` 紀錄。
4. 不可以假裝正式圖片、分層 PSD 或 Live2D rig 已經完成。
5. 缺圖時必須使用 placeholder fallback。
6. 不可以把角色寫成直接提供投資建議。
7. 不可以使用 API key、token、cookie、個資或私密憑證。
8. 第一版與 v2 工程都必須保持靜態網站架構，不依賴後端。
9. 每次有實質工作都要更新紀錄、狀態與 dashboard fallback data。
10. 專案名稱與根目錄固定為 `Minotaur-Grand-Mentor`。

## Obsidian 保護

禁止修改：

- `.obsidian/`
- `Daily Notes/`
- `Templates/`
- `Attachments/`
- 任何不是 `Minotaur-Grand-Mentor/` 的 vault 內容

## Live2D 邊界

Codex 可以完成 Live2D 規格、分層命名、rigger checklist、viewer 方向、驗收文件與 dashboard 狀態。Codex 不能憑空產出正式 PSD、`.moc3`、`model3.json` 或實際 rigging 成果。
